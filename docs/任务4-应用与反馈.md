# 任务4：系统应用与反馈

## 应用场景

本系统可应用于以下实际场景：

1. **二手车评估**: 快速确认车辆品牌型号，辅助估价
2. **交通管理**: 自动识别车型，用于交通流量统计
3. **停车场管理**: 车型识别辅助收费分级
4. **汽车资讯**: 用户拍照识别街上的陌生车型
5. **保险定损**: 快速确认车型信息，辅助理赔流程

## 用户反馈机制

### 前端反馈入口

每个模型的识别结果下方提供反馈按钮：

- **✓ 正确**: 确认该模型识别正确
- **✗ 有误**: 标记该模型识别错误，可补充正确标签

### 反馈数据存储

反馈数据以 JSONL 格式存储在 `backend/feedback.jsonl`：

```json
{
    "timestamp": "2026-07-01T14:30:00",
    "image_id": "a1b2c3d4e5f6",
    "model_id": "model_2",
    "is_correct": false,
    "correct_label": "BMW 3 Series Sedan 2012",
    "comment": "模型2把3系识别成了5系"
}
```

### 反馈驱动的改进流程

1. **收集阶段**: 用户上传图片→系统识别→用户点击反馈
2. **分析阶段**: 定期统计各模型准确率、高频误识别类别
3. **改进阶段**: 
   - 将误识别样本加入训练集重新训练
   - 针对性调整模型结构或超参数
   - 增加困难样本的数据增强

## 持续优化计划

1. **短期**: 收集 100+ 条用户反馈，分析各模型薄弱类别
2. **中期**: 针对 Top-10 易错类别专项优化（数据增强/难例挖掘）
3. **长期**: 支持用户提交标注图片扩充数据集，建立反馈闭环
